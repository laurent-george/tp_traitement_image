def format_coord(x, y, image):
    x = int(x + 0.5)
    y = int(y + 0.5)
    try:
        val = str(image[y, x])
    except IndexError:
        val = "?"
    return "color={} @ [{}, {}]".format(val, y, x)

def display_image_nb(image, title='', cmap=None):
    fig, ax = plt.subplots(1,1)
    if cmap is None:
        ax.matshow(image)
    else:
        ax.matshow(image, cmap=cmap)
    ax.set_xlabel(title)
    ax.format_coord = partial(format_coord, image=image)
    #return fig, ax

Analyse des performances.¶

La matrice de confusion permet visualiser la qualit d'un systme de classification.

Le nom "confusion" vient du fait que cette matrice permet facilement de voir si deux classes peuvent tre confondues (cd prdire une classe la place d'une autre).

  • ligne: classe attendue
  • colonne: classe prdite.

Dans le cas d'un systme parfait, toutes les valeurs se trouvent sur la diagonale.

Pour plus d'information: https://en.wikipedia.org/wiki/Confusion_matrix

Precision et rappel (recall)

rappel = True positive / (True positive + False negative)

prcision = True positive / (True positive + False positive)

Une prcision leve implique qu'on ne se trompe pas quand on prdit une valeur (autrement dit on est sur de nous quand on donne une valeur).

Un rappel lev signifie qu'on est capable de bien prdire un grand nombre des lments qui nous interessent.

Remarque, il est facile d'avoir une prcision leve si on a un rappel trs faible.

Cross validation

Apprendre sur des donnes et tester sur ces mmes donnes est une erreur mthodologique. Il faut toujours valuer les performances sur des donnes qui n'ont pas t utilises pour l'apprentissage. Cette approche ncessite d'avoir un jeu de donne d'apprentissage (X_learn, Y_learn) et un jeu de donne de test (X_test, Y_test).

Cependant partitionner les donnes disponnibles en deux ensembles rduit drastiquement le nombre de donnes utilisables pour apprendre le modle, de plus le rsultat pourrait tre dpendant du choix du partitionement.

Une solution est d'utiliser un schma de type cross-validation.

Lisez la documentation de cross_validation.cross_val_score et utilisez cette mthode.

k-fold cross validation:

  • partitionner les donnes en k sous ensembles pour i allant de 0 k:
    • apprendre sur les sous ensembles [0, .., i-1, i+1, .. k]
    • tester sur le jeu de donne i
  • Moyenner les rsultats

print(n)
X_train = digits.data[0:n]
Y_train = digits.target[0:n]
X_test = digits.data[n:]
Y_test = digits.target[n:]

gaussian.fit(X_train, Y_train)
